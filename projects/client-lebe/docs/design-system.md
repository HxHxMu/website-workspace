# Project Design System

## Rule
Each client project must define its own design system and visual language.

Do not assume the portfolio site's visual system applies here.

## Possible Design System Types
- custom brand system
- editorial marketing system
- conversion-focused landing page system
- product-style system
- Material Design 3 if appropriate

## Material Design Rule
Use Material Design only when the project benefits from:
- product/app-style interface behavior
- structured UI patterns
- familiar systemized interactions
- dashboard or application-like needs

Do not use Material automatically.

## Motion Rule
Motion is defined per project.

Default motion should be:
- clean
- subtle
- useful
- lightweight
