# Portfolio Design System

## Design System Type
Editorial Product Portfolio System

## Purpose
Create a polished, modern, premium-feeling portfolio site that presents Luis E. Bonilla as a strong product designer, visual designer, and creative technologist.

The site should feel like:
"a product designer with elite visual taste and strong systems thinking."

## Design Principles
- typography-led
- clean but expressive
- editorial clarity
- modern interface polish
- visual hierarchy first
- premium restraint
- high signal, low clutter
- portfolio should feel designed, not templated

## Visual Direction
The site should feel:
- thoughtful
- refined
- modern
- elevated
- creative but professional

Avoid:
- generic SaaS dashboard feel
- overly corporate blandness
- default Material-style UI
- over-designed visual noise
- templated portfolio aesthetics

## Layout Principles
Prefer:
- strong spacing
- large clean sections
- readable content width
- strong typography hierarchy
- selective emphasis
- modular reusable sections

## Typography Principles
Typography should lead the visual system.

Prioritize:
- strong headline hierarchy
- readable body copy
- clear section rhythm
- elegant contrast between large and small text

## Component Principles
Components should feel:
- clean
- intentional
- lightweight
- refined

Core component types:
- hero sections
- project cards
- section headers
- CTA blocks
- navigation
- image/media frames
- quote or insight blocks
- capability blocks

Avoid:
- overbuilt UI chrome
- app-like component heaviness
- excessive decorative elements

## Motion Principles
Use motion as polish, not decoration.

Preferred motion:
- hover transitions
- fade/reveal on entrance
- subtle scale/translate
- navigation feedback
- card interactivity
- smooth CTA behavior

Avoid:
- heavy motion libraries unless clearly needed
- excessive parallax
- distracting scroll gimmicks
- motion that competes with content

## Tailwind Usage
Tailwind is the implementation layer.

Use Tailwind for:
- layout
- spacing
- typography
- responsive design
- utility styling
- lightweight transitions

Do not let Tailwind dictate the visual identity.
The design system should lead; Tailwind should implement.
